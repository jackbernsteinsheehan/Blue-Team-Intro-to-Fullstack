Welcome to the API documentation. Essentially, this is a way of allowing you to call my functions from the frontend (it's how frontend talks to backend). There are a few endpoints that will be helpful to frontend people. They are ```extract_record``` and ```get_table_data```. In ```backend/blueprints/app.py```, you will find these endpoints, and you'll notice that the functions don't do anything, they just call functions (methods) from a different module. So if you need more detail on what they do, go to ```backend/database/Connection.py``` and find the definitions there.

```extract_record```: This is the function that you will call to get data to display. It's great because you can use filtering conditions. Here's an example of what these conditions could look like: ```"metric = 'open' AND mean > 0 AND mean < 10"```. If you want to get data on a specific company, you'll want to use ```ticker = '...'``` (replace with the ticker). If you want to allow users to filter based on some conditions, add them to your call to

```/extract/<ticker>/<field>/<conditions>/data``` # You can find these endpoints in the ```app.py``` file
. For example, calling the function with ```query_extract('test_metrics', "ticker, mean", "metric = 'open' AND mean > 0 AND mean < 10")``` will return ```[{'ticker': 'IBM', 'mean': 5.5}, {'ticker': 'IBM', 'mean': 5.5}, {'ticker': 'HEY', 'mean': 8.7}]``` for the test tables that I currently have. If you want, check out ```my test.py``` in the ```backend/database``` folder.

If you need help fioguring out how to hit these endpoints from the frontend, ask ChatGPT, or ask me.
